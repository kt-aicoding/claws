# OpenClaw 相关项目集合

这个目录用于在本机集中管理 OpenClaw 相关项目。各独立项目继续保留自己的 Git 仓库、远程地址和发布节奏；父仓库 `claws` 只跟踪本索引，不跟踪子目录内容。

## 仓库边界

- 在独立项目目录内执行 Git 命令，不要从 `claws` 父仓库批量提交子项目。
- `projects/*/` 已被父仓库忽略，避免把嵌套仓库误记为 gitlink，或把项目本地文件误提交到公开运维仓库。
- 更新某个产品时，遵循该目录内最近的 `AGENTS.md`、`CLAUDE.md` 和 `README.md`。
- 旧知识库在公开前必须单独检查绝对路径、账号标识、网络地址、精确排期、密钥模式、运行日志和 raw output。

## 未纳入项

- 原工作区中的 `openclaw/` 只有运行态备份压缩包，不是项目，也不符合本仓库的公开数据边界，因此没有迁入。
- OpenOctopus 等独立产品虽然同属 Agent 方向，但没有直接使用 OpenClaw 作为项目边界，因此没有按名称或关键词机械归入。
- `registry`、`agents/corpus` 等目录只保存索引或副本，不作为 OpenClaw 源项目重复迁移。
